    Capsanoto Guidebook

    1. What Capsanoto Is

    Capsanoto is a lightweight structured writing environment for lore-heavy writing, long-form worldbuilding, scripts, novels, pitch material, and linked narrative documents.

    It is built for human writing first: quick editing, readable files, local autosave, portable downloads, and simple structure without turning the app into a heavy database or wiki system.

    2. Why Capsanoto Exists

    Capsanoto exists because large writing projects often need more than a plain document, but less than a full database. A writer may need to reuse a paragraph, jump between character files, preserve older versions, insert tables, collect notes, or download clean copies without losing the flow of writing.

    The guiding idea is that structure should support writing, not take over the writing.

    3. Application Structure

    The intended hierarchy is:

    Application → Writing Room → Folder → Tab → File → TCard

    Application
    Capsanoto itself: the browser-based writing app.
    Writing Room
    The top-level container for one story world or major writing space.
    Folder
    A large category inside a Writing Room, such as Characters, Locations, Items, Worldbuilding, Songs, or Episodes.
    Tab
    A grouping inside a folder, such as Main Characters, Recurring Characters, Season 1, Lore Checks, or Drafts.
    File
    An individual writing document, such as Mel, Luca, Ironvale, Runestones, or Episode 01.
    TCard
    A Transclusion Card: a reusable source text block that can be referenced inside multiple files.

    4. Writing Room / Filing Cabinet

    The Writing Room button opens the Filing Cabinet, the floating Folder / Tab / File manager.

    Folders, Tabs, and Files
    Folders contain tabs. Tabs contain files. Files contain the actual writing.
    Edit mode
    The pencil unlocks inline editing for the Writing Room title, folder names, tab names, file names, and metadata values.
    Save layout
    When edit mode is on, the save-layout icon stores the Filing Cabinet position and size so it can reopen in the same place.
    Duplicate
    Creates a copy of a file for variants, rewrites, or experiments.
    Trash
    Deleted files move to Trash instead of disappearing immediately.
    Deprecated versions
    Deprecated versions are preserved older versions of a file. Archive wording should be reserved for preserved history, snapshots, retired files, and old versions.
    Jump rail
    The right-side rail uses folder icons as jump points. Collapse All and Expand All live at the top and bottom of the rail.

    5. Top Bar and Status Box

    The top bar contains the Capsanoto title, the Writing Room button, the center status box, and the main command icons.

    Status box
    The center status box explains hovered controls, cursor context, clicked-object context, save messages, autosave messages, and bottom document-button actions.
    Format
    Provides Normal Text, Bold, Italic, List, H1, H2, H3, and alignment controls.
    Insert
    Provides Image, Table, TCard, Horizontal Rule, Emphasis, and Emoji Spark.
    Subnoto
    Placeholder for a future clipboard panel.
    Specnoto
    Placeholder for a future search/find panel.
    Top Bar Height
    Settings → Design → System includes Top Bar Height so the main toolbar height can be adjusted while keeping controls vertically centered.

    6. Right-Click Context Tools

    Capsanoto is designed around selecting text and right-clicking it. The context menu keeps common structured-writing actions close to the writing surface.

    Link Anchor
    Turns selected text into a heading/bookmark target that can be linked directly.
    TCard
    Creates or references a Transclusion Card from selected text.
    Link
    Creates a normal hyperlink to a file, Link Anchor, TCard, or external URL.
    Link Pill
    Creates a rounded visual hyperlink.
    Link Editor
    Updates an existing link target without recreating the visible link text.
    Pill Color
    Changes the visual styling of a selected Link Pill.

    7. Links and Navigation

    Document links
    Files can be opened by URLs such as editor.html?doc=Character-Mel-Ameldra.
    Link Anchors
    Heading IDs create direct section routes such as editor.html?doc=Character-Mel-Ameldra#Runestones.
    External links
    Normal web URLs are supported.
    TCard references
    TCard targets can use references such as block:Item-Runestones or {{Item-Runestones}}.
    Link Pills
    Link Pills are rounded hyperlinks. Their visible text should stay clean and should not gain automatic arrow prefixes.

    8. TCards / Transclusion Cards

    TCard means Transclusion Card.

    A TCard stores reusable source text once and displays that same source text wherever the TCard is referenced. Transclusion is the behaviour of referencing that TCard elsewhere; it is not a separate standalone feature name.

    Syntax
    Use {{Item-Runestones}} to reference a TCard inside a file.
    Source text
    Editing the source TCard updates every rendered reference.
    Local styling
    A rendered TCard can have local visual styling while keeping the source text consistent.
    Insert behaviour
    Insert TCard places the reference at the saved cursor position.
    Drag/move
    Rendered TCards can be dragged inside the writing area.

    9. Tables, Emoji Spark, and Emphasis Boxes

    Tables
    Tables insert at the cursor, use rounded corners, can be dragged, and show inline tools for rows, columns, colours, borders, and alignment.
    Emoji Spark
    Emoji Spark opens an emoji picker. Click an emoji to insert it, right-click an emoji to copy/search depending on context, and use favorites for quick access.
    Emphasis boxes
    Emphasis boxes wrap selected text in a bordered block for notes, warnings, ideas, or highlighted story material.

    10. Document Editing and Autosave

    Document title
    The title field edits the current file title.
    Writing area
    The main editable page supports browser rich-text editing, headings, links, images, tables, TCards, and emphasis boxes.
    Local autosave
    Capsanoto currently saves locally in the browser using localStorage. This is the active safety layer.
    GitHub sync
    No GitHub sync engine is currently integrated in this browser app. Future sync should not block writing; if sync fails, local changes should remain safe.
    Status messages
    The status box should show messages such as Saving locally…, Autosaved locally, Syncing to GitHub…, Synced to GitHub, and Sync failed — saved locally when those systems exist.

    11. Download and Backups

    Download builder
    Choose files and/or TCards, reorder them, and download selected pieces.
    Caps file
    Downloads structured JSON for Capsanoto/Caps workflows.
    TXT
    Downloads plain text.
    HTML
    Downloads a browser-readable HTML file.
    Word-compatible
    Downloads a Word-compatible .doc HTML document.
    Clean output
    Downloads should render cleanly and strip editor-only UI artifacts.

    PDF and DOCX clean download are planned/experimental, not complete current features.

    12. Design Settings

    Settings → Design controls app styling such as fonts, colours, borders, shadows, top bar height, Writing Room window styling, Settings styling, Help styling, writing area controls, table defaults, TCard styling, and scrollbars where implemented.

    Design has three layers: Capsanoto default theme tokens in the code, the active Writing Room project theme saved in capsanoto.project.json, and temporary local unsaved changes while editing the settings. When you save design settings, they become part of the Writing Room project and will travel with HDD backup and Google Drive sync.

    Recommended font stack: Arial, Helvetica, sans-serif. Courier New is reserved for code-like/monospace text.

    Command Layout Editor

    Subnoto

    14. Technical Notes

    Code type
    Static HTML, CSS, and plain JavaScript. No framework build step is required.
    Main app file
    app.js handles state, events, local autosave, routing, TCards, links, downloads, dialogs, Filing Cabinet behaviour, table tools, Emoji Spark, and design settings.
    Styles
    styles.css controls the theme, layouts, panels, typography, and CSS variables.
    Starter files
    content/documents.json seeds first-run content. config/auth.json remains for future preview auth work.
    Local storage keys
    forever-bound-writing-room-v2 stores files/TCards, capsanoto-design-settings-v5-clean stores design settings, capsanoto-help-html-v1 stores edited Help text, and capsanoto-writing-room-layout-v1 stores Filing Cabinet position/size, and selected deprecated paragraph history is stored inside the Writing Room data.
    Save audit
    Current save behaviour is local-first. File edits, TCard edits, Filing Cabinet changes, and design settings save to browser localStorage. A durable GitHub sync system is not currently present in the app code.
    Versioning rule
    Use v1 as the first version, then v1.1, v1.2, and so on for future updates.
